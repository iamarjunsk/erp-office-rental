## 2025-03-17 - Use aggregate and Count with Q objects for faster stats endpoints **Learning:** I learned that making multiple `.count()` queries sequentially on the same queryset can quickly cause performance degradation and excess round trips. Django provides a great mechanism for condensing this into a single query. **Action:** Next time I encounter multiple counts or stats-gathering on a single endpoint, I will use `.aggregate()` combined with `Count('id', filter=Q(...))` to calculate all metrics in a single database round trip.
