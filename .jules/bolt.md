## 2024-05-24 - N+1 query bottleneck in nested Django REST Framework serializers
**Learning:** Even if `prefetch_related` is used on a ModelViewSet (`prefetch_related('tasks', 'comments')`), if the nested serializers for those related items fetch further foreign keys (like `assigned_to` on a Task or `author` on a Comment), DRF will still execute an additional query per item unless `Prefetch` with `select_related` is used. This can cause the query count to explode from e.g. 3 to 33 in just 5 items.
**Action:** When using `prefetch_related` on a ViewSet, check the serializers of the prefetched models. If they access foreign key attributes via nested serializers, use `django.db.models.Prefetch('relation_name', queryset=RelatedModel.objects.select_related('nested_fk'))` to efficiently load the nested foreign keys in the same query.
