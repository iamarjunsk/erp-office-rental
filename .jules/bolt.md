## 2025-02-13 - Optimize API stats queries with aggregate(Count(filter=Q(...))) **Learning:** Using `.count()` and `.aggregate()` separately for different dashboard metrics (e.g. status counts) results in 5-10 separate database queries. These can all be condensed into a single database hit using Django's conditional aggregation `Count('pk', filter=Q(condition))` and `Sum('field', filter=Q(condition))`. **Action:** When reviewing dashboard or stats endpoints, always check if multiple simple counts/sums can be batched into a single `.aggregate()` call to reduce query latency and DB roundtrips.
