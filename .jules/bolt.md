## 2024-03-22 - [Fix N+1 query in BookingViewSet] **Learning:** DRF nested serializers for ManyToOne or OneToOne fields (like `space_details` and `company_details` in `BookingSerializer`) cause severe N+1 query problems if the ViewSet's `queryset` is not optimized with `select_related`. **Action:** Always inspect serializers for nested relationships and use `select_related` on the corresponding ViewSet `queryset` to fetch them efficiently in a single SQL query. Add `assertNumQueries` in tests to prevent regressions.
