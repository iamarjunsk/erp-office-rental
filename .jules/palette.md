## 2024-03-25 - Replace native alerts with shadcn toast | Learning: The codebase was using native `alert()` for displaying backend API validation errors and informational messages, causing a blocking and unstyled user experience. Shadcn's Toast component provides a non-blocking `variant: 'destructive'` for errors. | Action: Replaced all 19 occurrences of `alert()` across the admin pages with `useToast().toast(...)` and improved the formatting of joined error messages to use commas instead of newlines for better inline readability within the toast description.
